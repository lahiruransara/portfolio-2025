Overview
CORONA Admin Unofficial Design System
Corona Admin template flaunts a simple and unique design along with an intriguing dark and shadowy color scheme. It contains a powerful dashboard and is packed with features and useful components capable of simplifying complex web app development projects. Data representation and visualization are no longer challenging, as the template offers multiple options for the same.
Research
Learning the basics of a design system
The architecture I chose to follow is called The Atomic Design Approach. I picked this method because of its logic and how it wil help me to organize these components by complexity and find them quickly.

Define & Ideation
During the creation of each component, the following concepts were followed:
  • Responsive components with the auto-layout feature to ensure the component could be reused for different devices or layouts.
  • Cover all basic states: hover, default, active, error, and disabled states.
  • Scalable component patterns with a focus on maintaining an organized library to ensure it remains perfectly in sync with the design and code disciplines.
  • Clear naming conventions to ensure design and code disciplines were aligned.

Next Steps
Looking towards the future
Through this case study, I wanted to share with you my process and a bit of what I learned along the way. Like many things, a design system isn’t ever a finished thing — it’s a journey. So far, the set of basic components has been game-changing for me.
As the design system grows, I hope to create more standardized assets that can be reused in many ways. I hope to break down the the components and document them with how the component behaves.
